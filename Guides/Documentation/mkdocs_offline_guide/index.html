    MkDocs Offline Documentation Setup Guide¶

    This guide walks you through a full offline-first setup of MkDocs with Material theme, including customization with CSS and project structure for technical documentation.

    üöÄ Installation Steps (Ubuntu/Debian)¶

    1. Install Python, pip, and virtualenv:
    sudo apt update
sudo apt install -y python3 python3-pip python3-venv git
    1. Create project folder and virtual environment:
    mkdir ~/my-docs
cd ~/my-docs
python3 -m venv .venv
source .venv/bin/activate
    1. Install MkDocs and Material theme:
    pip install mkdocs-material

    ‚öôÔ∏è MkDocs + Material Setup¶

    Edit mkdocs.yml:

    site_name: My Offline Docs

theme:
  name: material
  palette:
    scheme: slate
  font:
    text: Roboto Mono
    code: JetBrains Mono

extra_css:
  - assets/css/custom.css

nav:
  - Home: index.md
  - Introduction: intro.md
  - Projects:
      - Cloud: cloud.md
      - AI: ai.md
      - DevOps: devops.md

    üé® Custom Theme with CSS (Optional)¶

    Place this in docs/assets/css/custom.css:

    /* Background for homepage */
body[data-md-page="index"] .md-main__inner.md-grid {
    background-image: url("../images/bg.jpg");
    background-size: cover;
    background-position: center;
    background-repeat: no-repeat;
    min-height: 100vh;
}

/* Cool tech font + color scheme */
body {
    font-family: 'Fira Code', monospace;
    color: #40E0D0;
}

/* Transparent background boxes */
.md-content, .md-sidebar, .md-header {
    background-color: transparent !important;
}

.md-header {
    background-color: #012a2a !important;
}

    üß™ Preview the Site Offline¶

    Run locally with hot reload:

    mkdocs serve

    Visit: http://127.0.0.1:8000

    üèó Build Static Site (for offline use)¶

    mkdocs build

    This creates a static site in the site/ directory. You can open site/index.html directly in any browser‚Äîno server needed.

    To copy to another machine:

    tar -czvf my-docs.tar.gz site/

    ‚òÅÔ∏è (Optional) Deploy to GitHub Pages¶
